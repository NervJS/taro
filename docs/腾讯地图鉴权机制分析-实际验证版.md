# 腾讯地图 JavaScript API 鉴权机制分析 (实际验证版)

## 一、核心发现

### 1.1 鉴权失败的日志

通过实际测试,使用无效 key (`OB4BZ-D4W3U-B7VVO-4PJWW-6TKDJ-WPB77000`) 时,日志如下:

```
init==================3 undefined
地图初始化成功 e {...}
onAuthSuccess:res {type: 'authsuccess', ...}  ← onAuthSuccess 先触发
===========>error 鉴权失败：KEY_FORMAT_ERROR  ← console.error 报错!
鉴权失败：海外图鉴权失败  ← 第二个错误
```

### 1.2 关键发现

**无论鉴权成功还是失败,地图 UI 都能正常展示,唯一的区别是日志!**

腾讯地图 SDK 在鉴权失败时会:
1. **调用 `console.error`** 输出 `鉴权失败：XXX`
2. **地图初始化仍会继续**,只是功能受限

### 1.3 错误信息类型

从日志可以看到两种错误:
- `鉴权失败：KEY_FORMAT_ERROR` - key 格式错误
- `鉴权失败：海外图鉴权失败` - key 与域名不匹配

## 二、正确的检测方案

### 2.1 方案: 监听 console.error

```javascript
function useMapLoader({ onAuthSuccess, onError, id }) {
  const [status, setStatus] = useState<'loading' | 'success' | 'error'>('loading')
  const originalErrorRef = useRef(null)
  const authFailedRef = useRef(false)
  const errorMsgRef = useRef('')

  useEffect(() => {
    // 1. 保存原始 console.error
    originalErrorRef.current = console.error

    // 2. 拦截 console.error
    console.error = function(...args) {
      const message = args.join(' ')

      // 3. 检查是否是鉴权失败
      if (message.includes('鉴权失败')) {
        console.log('❌ 检测到鉴权失败:', message)
        authFailedRef.current = true
        errorMsgRef.current = message

        // 触发 onError
        if (typeof onError === 'function') {
          onError({
            type: 'error',
            timeStamp: Date.now(),
            target: { id: id || '', tagName: 'map', dataset: {} },
            currentTarget: { id: id || '', tagName: 'map', dataset: {} },
            detail: {
              longitude: 0,
              latitude: 0,
              errMsg: message,
              errCode: getErrorCode(message),
            },
            preventDefault: () => {},
            stopPropagation: () => {},
          })
        }
      }

      // 4. 调用原始 console.error
      originalErrorRef.current?.apply(console, args)
    }

    return () => {
      // 5. 恢复原始 console.error
      if (originalErrorRef.current) {
        console.error = originalErrorRef.current
      }
    }
  }, [onError, id])

  // 6. 地图初始化成功回调
  const  = (mapInstance) => {
    console.log('地图初始化成功,检查鉴权状态...')

    // 如果之前已经检测到鉴权失败,不触发 onAuthSuccess
    if (authFailedRef.current) {
      console.log('❌ 鉴权失败,地图虽有初始化但功能受限')
      return
    }

    // 鉴权成功
    console.log('✅ 鉴权成功')
    setStatus('success')
    if (typeof onAuthSuccess === 'function') {
      onAuthSuccess({
        type: 'authsuccess',
        timeStamp: Date.now(),
        target: { id: id || '', tagName: 'map', dataset: {} },
        currentTarget: { id: id || '', tagName: 'map', dataset: {} },
        detail: { errCode: 0, errMsg: 'ok' },
        preventDefault: () => {},
        stopPropagation: () => {},
      })
    }
  }

  return { status, handleMapInited }
}

// 错误码映射
function getErrorCode(message: string): number {
  if (message.includes('KEY_FORMAT_ERROR')) {
    return 1001 // key 格式错误

  if (message.includes('海外图鉴权失败')) {
```

### 2.2 完整实现

```tsx
// packages/taro-components-react/src/components/map/index.tsx

import { useEffect, useRef, useState } from 'react'

const MAP_INIT_TIMEOUT = 15000

// 错误码映射
function getAuthErrorCode(message: string): number {
  if (message.includes('KEY_FORMAT_ERROR')) {
    return 1001 // key 格式错误
  }
  if (message.includes('海外图鉴权失败')) {
    return 1002 // key 与域名不匹配
  }
  return 1003 // 其他错误
}

export function useMapAuth({
  onAuthSuccess,
  onError,
  id,
}: {
  onAuthSuccess?: (e: any) => void
  onError?: (e: any) => void
  id?: string
}) {
  const [authStatus, setAuthStatus] = useState<'loading' | 'success' | 'error'>('loading')
  const originalErrorRef = useRef<((...args: any[]) => void) | null>(null)
  const authFailedRef = useRef(false)
  const timeoutRef = useRef<NodeJS.Timeout>()

  useEffect(() => {
    // 1. 保存原始 console.error
    originalErrorRef.current = console.error.bind(console)

    // 2. 拦截 console.error
    console.error = ((...args: any[]) => {
      const message = String(args[0] || '')

      // 3. 检查是否是鉴权失败
      if (message.includes('鉴权失败')) {
        console.log('[Map] ❌ 鉴权失败:', message)
        authFailedRef.current = true

        // 清除超时
        if (timeoutRef.current) {
          clearTimeout(timeoutRef.current)
        }

        setAuthStatus('error')

        // 触发 onError
        if (typeof onError === 'function') {
          onError({
            type: 'error',
            timeStamp: Date.now(),
            target: { id: id || '', tagName: 'map', dataset: {} },
            currentTarget: { id: id || '', tagName: 'map', dataset: {} },
            detail: {
              longitude: 0,
              latitude: 0,
              errMsg: message,
              errCode: getAuthErrorCode(message),
            },
            preventDefault: () => {},
            stopPropagation: () => {},
          })
        }
      }

      // 4. 调用原始 console.error
      originalErrorRef.current?.(...args)
    }) as typeof console.error

    // 5. 设置超时
    timeoutRef.current = setTimeout(() => {
      if (authStatus === 'loading') {
        console.log('[Map] ⚠️ 地图加载超时')
        setAuthStatus('error')
        if (typeof onError === 'function') {
          onError({
            type: 'error',
            timeStamp: Date.now(),
            target: { id: id || '', tagName: 'map', dataset: {} },
            currentTarget: { id: id || '', tagName: 'map', dataset: {} },
            detail: {
              longitude: 0,
              latitude: 0,
              errMsg: '地图加载超时,请检查网络连接',
              errCode: 1004,
            },
            preventDefault: () => {},
            stopPropagation: () => {},
          })
        }
      }
    }, MAP_INIT_TIMEOUT)

    return () => {
      // 6. 恢复原始 console.error
      if (originalErrorRef.current) {
        console.error = originalErrorRef.current
      }
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current)
      }
    }
  }, [onError, id])

  // 7. 地图初始化成功回调
  const handleMapInited = (mapInstance: any) => {
    console.log('[Map] 地图初始化成功,检查鉴权状态...')

    // 如果之前已经检测到鉴权失败,不触发 onAuthSuccess
    if (authFailedRef.current) {
      console.log('[Map] ⚠️ 鉴权失败,地图虽有初始化但功能受限')
      return
    }

    // 清除超时
    if (timeoutRef.current) {
      clearTimeout(timeoutRef.current)
    }

    console.log('[Map] ✅ 鉴权成功')
    setAuthStatus('success')

    if (typeof onAuthSuccess === 'function') {
      onAuthSuccess({
        type: 'authsuccess',
        timeStamp: Date.now(),
        target: { id: id || '', tagName: 'map', dataset: {} },
        currentTarget: { id: id || '', tagName: 'map', dataset: {} },
        detail: { errCode: 0, errMsg: 'ok' },
        preventDefault: () => {},
        stopPropagation: () => {},
      })
    }
  }

  return { authStatus, handleMapInited }
}
```

## 三、错误码定义

| 错误码 | 含义 | 触发条件 |
|--------|------|----------|
| 1001 | KEY 格式错误 | `鉴权失败：KEY_FORMAT_ERROR` |
| 1002 | key 与域名不匹配 | `鉴权失败：海外图鉴权失败` |
| 1003 | 其他鉴权错误 | 其他包含 "鉴权失败" 的错误 |
| 1004 | 地图加载超时 | 15秒内未完成初始化 |

## 五、总结

### 核心要点

1. **地图 UI 正常展示 ≠ 鉴权成功**: 无论成功失败,UI 都能显示
2. **日志是唯一区别**: 鉴权失败时 `console.error` 输出 "鉴权失败：XXX"
3. **监听 console.error**: 通过拦截 `console.error` 检测鉴权状态

### 实现优势

- ✅ **准确**: 直接监听腾讯 SDK 的错误输出
- ✅ **实时**: 错误发生时立即检测
- ✅ **完整**: 覆盖所有类型的鉴权失败
- ✅ **简单**: 不需要额外创建测试地图实例

### 注意事项

- ⚠️ 拦截 `console.error` 是全局的,需要及时恢复
- ⚠️ 需要在脚本加载前就设置拦截
- ⚠️ 地图初始化成功回调需要检查是否已检测到失败
