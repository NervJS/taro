# 腾讯地图 JavaScript API 鉴权机制分析 (最终版)

## 一、核心发现

### 1.1 脚本对比结果

通过对比三个脚本文件:

| 文件 | key 值 | 说明 | init调用 |
|------|--------|------|---------------|
| success.js | `OB4BZ-...WPB77` (真实) | ✅ 有效 key | ✅ 有 |
| fail.js | `""` (空) | ❌ key 为空 | ✅ 有 |
| fail2.js | `OB4BZ-...000` (假的) | ❌ key 无效 | ✅ 有 |

### 1.2 关键结论

**无论 key 是否有效,脚本都会调用 `window.init()`!**

这意味着:
- ❌ **无法通过监听 `init` 是否被调用来判断鉴权是否成功**
- ✅ **真正的鉴权检查发生在运行时**,当实际使用地图 API 时才会知道

### 1.3 鉴权流程

```
1. 加载脚本 (无论 key 是否有效,都会成功返回)
   ↓
2. 脚本末尾调用 window.init()
   ↓
3. TMap.__load 返回 { key: "xxx" } (key 可能有值也可能为空)
   ↓
4. 尝试创建 new TMap.Map() ← 这里是真正的鉴权检查
   ↓
   ┌──────────────┬──────────────┐
   │  成功创建     │   创建失败    │
   │  ✅ 鉴权成功  │  ❌ 鉴权失败  │
   └──────────────┴──────────────┘
```

## 二、正确的检测方案

### 2.1 方案: 监听 init+ 实际创建地图验证

```javascript
function useMapLoader({ authKey, onSuccess, onError }) {
  useEffect(() => {
    let timeoutId
    let initCalled = false

    // 1. 监听 init判断脚本是否加载
    const originalInit = window.init
    window.init= () => {
      initCalled = true
      clearTimeout(timeoutId)

      console.log('✅ 脚本加载完成,尝试创建地图实例...')

      // 2. 实际创建地图实例来验证鉴权
      try {
        const container = document.createElement('div')
        container.style.display = 'none'
        document.body.appendChild(container)

        const testMap = new TMap.Map(container, {
          center: new TMap.LatLng(39.984104, 116.307503),
          zoom: 12,
        })

        // 3. 如果成功创建,说明鉴权通过
        console.log('✅ 地图实例创建成功,鉴权通过')
        testMap.destroy()
        document.body.removeChild(container)

        onSuccess()
      } catch (error) {
        // 4. 如果创建失败,说明鉴权失败
        console.error('❌ 地图实例创建失败,鉴权失败:', error)
        document.body.removeChild(container)

        onError('鉴权失败,请检查 API key 是否正确')
      }

      // 调用原始回调
      if (originalInit) {
        originalInit()
      }
    }

    // 5. 设置超时 - 处理网络错误
    timeoutId = setTimeout(() => {
      if (!initCalled) {
        console.error('❌ 脚本加载超时')
        onError('地图脚本加载超时,请检查网络连接')
      }
    }, 10000)

    return () => {
      clearTimeout(timeoutId)
      if (originalInit) {
        window.init= originalInit
      } else {
        delete window.init
      }
    }
  }, [])
}
```

### 2.2 完整实现

```tsx
// packages/taro-components-react/src/components/map/index.tsx

import { useEffect, useRef, useState } from 'react'

const MAP_INIT_TIMEOUT = 15000 // 15秒超时

function useMapLoader({
  authKey,
  onAuthSuccess,
  onError,
  latitude,
  longitude,
  id,
}: {
  authKey?: string
  onAuthSuccess?: (e: any) => void
  onError?: (e: any) => void
  latitude?: number
  longitude?: number
  id?: string
}) {
  const [status, setStatus] = useState<'loading' | 'success' | 'error'>('loading')
  const timeoutRef = useRef<NodeJS.Timeout>()
  const originalInitRef = useRef<(() => void) | null>(null)

  const createErrorEvent = (errMsg: string, errCode: number) => ({
    type: 'error' as const,
    timeStamp: Date.now(),
    target: {
      id: id || '',
      tagName: 'map',
      dataset: {},
    },
    currentTarget: {
      id: id || '',
      tagName: 'map',
      dataset: {},
    },
    detail: {
      longitude: longitude || 0,
      latitude: latitude || 0,
      errMsg,
      errCode,
    },
    preventDefault: () => {},
    stopPropagation: () => {},
  })

  useEffect(() => {
    // 如果 TMap 已存在,直接验证
    if (window.TMap) {
      verifyMapCreation()
      return
    }

    // 保存原始回调
    if (window.init) {
      originalInitRef.current = window.init
    }

    // 设置我们的回调
    window.init= () => {
      console.log('脚本加载完成,开始验证地图...')

      // 清除超时
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current)
      }

      // 验证地图创建
      verifyMapCreation()

      // 调用原始回调
      if (originalInitRef.current) {
        originalInitRef.current()
      }
    }

    // 设置超时
    timeoutRef.current = setTimeout(() => {
      console.error('脚本加载超时')
      setStatus('error')
      if (typeof onError === 'function') {
        onError(createErrorEvent('地图脚本加载超时,请检查网络连接', 1001))
      }
    }, MAP_INIT_TIMEOUT)

    return () => {
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current)
      }
      if (originalInitRef.current) {
        window.init= originalInitRef.current
      } else {
        delete window.init
      }
    }

    // 验证地图创建
    function verifyMapCreation() {
      try {
        // 创建隐藏的容器
        const container = document.createElement('div')
        container.style.cssText = 'position:absolute;visibility:hidden;width:1px;height:1px;'
        document.body.appendChild(container)

        // 尝试创建地图实例
        const testMap = new TMap.Map(container.id || 'verify-map', {
          center: new TMap.LatLng(
            typeof latitude === 'number' ? latitude : 39.984104,
            typeof longitude === 'number' ? longitude : 116.307503
          ),
          zoom: 12,
        })

        console.log('✅ 地图创建成功,鉴权通过')

        // 清理测试容器
        setTimeout(() => {
          testMap.destroy()
          document.body.removeChild(container)
        }, 100)

        setStatus('success')
        if (typeof onAuthSuccess === 'function') {
          onAuthSuccess({
            type: 'authsuccess',
            timeStamp: Date.now(),
            target: {
              id: id || '',
              tagName: 'map',
              dataset: {},
            },
            currentTarget: {
              id: id || '',
              tagName: 'map',
              dataset: {},
            },
            detail: {
              errCode: 0,
              errMsg: 'ok',
            },
            preventDefault: () => {},
            stopPropagation: () => {},
          })
        }
      } catch (error: any) {
        console.error('❌ 地图创建失败,鉴权失败:', error)
        setStatus('error')
        if (typeof onError === 'function') {
          onError(createErrorEvent(
            error?.message || '地图创建失败,请检查 API key 是否正确',
            1002
          ))
        }
      }
    }
  }, [authKey, onAuthSuccess, onError, latitude, longitude, id])

  return status
}
```

## 三、错误码定义

| 错误码 | 含义 | 触发条件 |
|--------|------|----------|
| 1001 | 地图脚本加载超时 | 15秒内脚本未加载 (网络问题) |
| 1002 | 地图初始化失败 (鉴权失败) | `new TMap.Map()` 抛出异常 |

## 五、总结

### 核心要点

1. **脚本加载 ≠ 鉴权成功**: 无论 key 是否有效,脚本都会加载成功
2. **创建地图 = 真正鉴权**: 只有实际创建地图实例才能确认 key 是否有效
3. **超时处理网络错误**: 通过超时检测脚本加载失败

### 推荐实现

✅ **监听 init+ 创建地图实例验证 + 超时检测**

- ✅ 能准确检测鉴权状态
- ✅ 能处理网络错误
- ✅ 兼容性好
- ✅ 逻辑严密
