# 腾讯地图 JavaScript API 鉴权机制分析 (修正版)

## 一、脚本加载流程

### 1.1 加载 URL

```
https://map.qq.com/api/gljs?v=1.exp&key=YOUR_API_KEY&callback=init
```

**参数说明**:
- `v`: 版本号 (如 1.exp)
- `key`: 开发者密钥 (API Key)
- `callback`: 回调函数名 (tlbs-map-react 使用 `init`)

### 1.2 加载过程

```
浏览器请求脚本
    ↓
腾讯服务器验证 API Key
    ↓
┌─────────────┬─────────────┐
│  Key 有效   │  Key 无效   │
└─────────────┴─────────────┘
      ↓              ↓
返回完整脚本      返回完整脚本
key: "真实key"     key: ""
      ↓              ↓
都调用 init()    都调用 init()
      ↓              ↓
  需要检查 key   需要检查 key
```

## 二、脚本结构分析 (重要发现)

### 2.1 实际对比结果

通过对比 `success.js` 和 `fail.js` 两个文件:

**文件大小**: 完全相同 (224056 行)

**关键差异**:

```javascript
// 第 5 行 - TMap.__load 回调
// ✅ 鉴权成功
onLoad({
    "key": "OB4BZ-D4W3U-B7VVO-4PJWW-6TKDJ-WPB77",  // ← 有值
    "version": "1.8.0.2",
    ...
})

// ❌ 鉴权失败
onLoad({
    "key": "",  // ← 空字符串
    "version": "1.8.0.2",
    ...
})
```

**最后一行**: 完全相同
```javascript
// ✅ 鉴权成功
window.init&& window.init();

// ❌ 鉴权失败
window.init&& window.init();
```

### 2.2 重大发现

**无论鉴权成功还是失败,脚本末尾都会调用 `window.init()`!**

这意味着:
- ❌ **不能通过监听 `window.init` 是否被调用来判断鉴权**
- ✅ **需要检查 `TMap.__load` 回调返回的 `key` 字段**

## 三、如何判断鉴权成功/失败

### 3.1 判断依据

| 状态 | init 回调 | TMap.__load 的 key | 判断方法 |
|------|-----------|-------------------|----------|
| 鉴权成功 | ✅ 被调用 | `"真实key"` | 检查 key 非空 |
| 鉴权失败 | ✅ 被调用 | `""` (空) | 检查 key 为空 |
| 网络错误 | ❌ 未调用 | N/A | 超时检测 |

### 3.2 正确的判断方法

**方法1: 拦截 TMap.__load 回调**

```javascript
// 在加载脚本前拦截
const originalLoad = TMap.__load

TMap.__load = function(onLoad) {
  return function(config) {
    console.log('脚本加载完成,API Key:', config.key)

    if (!config.key || config.key === '') {
      console.error('❌ 鉴权失败: key 为空')
      onError('鉴权失败')
    } else {
      console.log('✅ 鉴权成功: key =', config.key)
      onSuccess()
    }

    // 调用原始回调
    if (originalLoad) {
      originalLoad(onLoad)(config)
    }
  }
}
```

**方法2: 监听 init 回调 + 检查 key**

```javascript
let scriptLoaded = false

// 设置 init 回调
window.init= function() {
  scriptLoaded = true
  console.log('init 被调用')

  // 检查 TMap.__load 返回的 key
  setTimeout(() => {
    TMap.__load(function(config) {
      if (!config.key || config.key === '') {
        console.error('❌ 鉴权失败')
        onError('鉴权失败')
      } else {
        console.log('✅ 鉴权成功')
        onSuccess()
      }
    })
  }, 100)
}

// 设置超时检测
setTimeout(() => {
  if (!scriptLoaded) {
    console.error('❌ 网络错误,脚本未加载')
    onError('网络错误')
  }
}, 10000)
```

## 四、tlbs-map-react 的实现问题

### 4.1 当前实现

```javascript
// tlbs-map-react/src/TMap/index.tsx

const loadScript = async (key: string, libraries?: string) => new Promise((resolve) => {
  if (window.TMap) {
    resolve(true);
    return;
  }

  resolveArr.push(resolve);

  if (!window.init) {
    window.init = () => {
      resolveArr.forEach((resolve) => resolve(true));  // ⚠️ 直接返回成功
    };
  }

  // ... 加载脚本
});
```

**问题**:
1. ❌ `init` 回调被调用时,直接返回 `true`
2. ❌ 没有检查 `TMap.__load` 返回的 `key` 字段
3. ❌ 无法区分鉴权成功和鉴权失败

### 4.2 为什么 tlbs-map-react 没有检测鉴权失败?

因为:
- 腾讯地图的脚本无论鉴权成功还是失败,都会调用 `init` 回调
- 唯一的区别是 `TMap.__load` 回调中的 `key` 字段
- tlbs-map-react 没有检查这个字段

## 五、我们的解决方案

### 5.1 核心思路

**监听 `window.init` 回调 + 检查 `TMap.__load` 的 key 字段 + 超时检测**

```javascript
function useMapLoader({ authKey, onSuccess, onError }) {
  useEffect(() => {
    let timeoutId
    let initCalled = false

    // 拦截 TMap.__load
    const originalObj = window.TMap || {}
    let interceptedConfig = null

    // 重新定义 TMap.__load
    Object.defineProperty(window.TMap || (window.TMap = {}), '__load', {
      get() {
        return function(onLoad) {
          return function(config) {
            interceptedConfig = config
            console.log('TMap.__load called with key:', config.key)

            // 调用原始 onLoad
            onLoad(config)
          }
        }
      },
      set(value) {
        originalObj.__load = value
      }
    })

    // 设置 init 回调
    const originalInit = window.init
    window.init = () => {
      initCalled = true
      clearTimeout(timeoutId)

      console.log('init called, intercepted config:', interceptedConfig)

      // 检查 key
      if (!interceptedConfig || !interceptedConfig.key || interceptedConfig.key === '') {
        console.error('鉴权失败: key 为空')
        onError('鉴权失败,请检查 API key')
      } else {
        console.log('鉴权成功: key =', interceptedConfig.key)
        onSuccess()
      }

      // 调用原始回调
      if (originalInit) {
        originalInit()
      }
    }

    // 设置超时
    timeoutId = setTimeout(() => {
      if (!initCalled) {
        console.error('脚本加载超时')
        onError('地图脚本加载超时,请检查网络连接')
      }
    }, 10000)

    return () => {
      clearTimeout(timeoutId)
      // 恢复原始回调
      if (originalInit) {
        window.init = originalInit
      } else {
        delete window.init
      }
    }
  }, [])
}
```

### 5.2 简化方案

如果拦截 `TMap.__load` 太复杂,可以使用更简单的方法:

**方案: 直接检查最终使用的 key**

```javascript
function useMapLoader({ authKey, onSuccess, onError }) {
  useEffect(() => {
    let timeoutId
    let initCalled = false

    window.init = () => {
      initCalled = true
      clearTimeout(timeoutId)

      // 触发 TMap.__load 回调来获取配置
      setTimeout(() => {
        // 尝试创建地图实例来验证
        try {
          // 如果 key 无效,这里会失败
          const testDiv = document.createElement('div')
          const testMap = new TMap.Map(testDiv, {
            center: new TMap.LatLng(39.984104, 116.307503),
            zoom: 12
          })

          // 如果成功创建,说明鉴权通过
          testMap.destroy()
          onSuccess()
        } catch (error) {
          console.error('创建地图失败:', error)
          onError('鉴权失败,请检查 API key')
        }
      }, 100)
    }

    timeoutId = setTimeout(() => {
      if (!initCalled) {
        onError('地图脚本加载超时')
      }
    }, 10000)

    return () => {
      clearTimeout(timeoutId)
    }
  }, [])
}
```

## 六、推荐方案

考虑到实现复杂度,推荐使用**超时检测 + 实际创建地图验证**的方案:

### 优点
1. ✅ 简单可靠
2. ✅ 能检测网络错误
3. ✅ 能检测鉴权失败
4. ✅ 兼容性好

### 缺点
1. ⚠️ 需要实际创建地图实例
2. ⚠️ 会增加一些初始化时间

## 七、总结

### 核心发现

1. **鉴权成功和失败都会调用 `init`**: 不能通过监听回调判断
2. **key 字段是唯一区别**: 成功时有值,失败时为空
3. **tlbs-map-react 没有检测**: 直接认为 `init` 被调用就是成功

### 推荐实现

✅ **超时检测 + 创建地图实例验证**

- 简单可靠
- 能准确检测鉴权状态
- 兼容性好
