# 腾讯地图鉴权状态判断方案

## 一、问题背景

在封装 Taro Map 组件时，需要判断腾讯地图 SDK 的鉴权状态（成功/失败），以便触发 `onAuthSuccess` 或 `onError` 回调。

## 二、测试发现

### 2.1 tilesloaded 和 idle 事件不可靠

从测试日志发现，即使 Key 错误，地图仍然会触发 `tilesloaded` 和 `idle` 事件：

```
失败：KEY_FORMAT_ERROR
鉴权失败：海外图鉴权失败
=================>瓦片加载完成,地图真正可用  ✅ 仍然触发
=================>地图已就绪  ✅ 仍然触发
```

**结论**：这两个事件无法用于判断鉴权状态。

### 2.2 腾讯地图的错误输出方式

腾讯地图 SDK 通过 `console.error` 输出鉴权错误信息：

```
失败：KEY_FORMAT_ERROR
鉴权失败：海外图鉴权失败
```

## 三、可能的错误类型

| 错误类型 | 是否有日志 | 日志内容 |
|---------|----------|---------|
| KEY格式错误 | ✅ 有日志 | KEY_FORMAT_ERROR |
| KEY不存在 | ❓ 未知 | ❓ 未知 |
| KEY过期 | ❓ 未知 | ❓ 未知 |
| 配额用尽 | ❓ 未知 | ❓ 未知 |
| 域名限制 | ❓ 未知 | ❓ 未知 |
| 网络错误 | ❓ 未知 | ❓ 未知 |
| 服务端异常 | ❓ 未知 | ❓ 未知 |

**已知错误关键词**：
- `鉴权失败`
- `KEY`
- `失败`
- `KEY_FORMAT_ERROR`
- `海外图鉴权失败`

## 四、mapInstance 可监听的事件

TMap.Map 实例支持以下事件（通过 `mapInstance.on(eventName, callback)` 监听）：

### 核心事件
| 事件名 | 说明 |
|--------|------|
| `idle` | 地图进入空闲状态时触发 |
| `tilesloaded` | 当地图容器中可见的瓦片加载完后触发 |
| `context_lost` | 地图渲染上下文丢失时触发 |
| `destroyed` | 地图被完全销毁时触发 |

### 鼠标/触摸事件
| 事件名 | 说明 |
|--------|------|
| `click` | 鼠标左键单击 |
| `rightclick` | 鼠标右键单击(仅桌面) |
| `dblclick` | 鼠标左键双击 |
| `mousedown/mouseup/mousemove` | 鼠标事件(仅桌面) |
| `touchstart/touchmove/touchend` | 触摸事件(仅移动) |

### 地图操作事件
| 事件名 | 说明 |
|--------|------|
| `dragstart/drag/dragend` | 拖拽事件 |
| `panstart/pan/panend` | 平移事件 |
| `rotatestart/rotate/rotateend` | 旋转事件 |
| `pitchstart/pitch/pitchend` | 俯仰事件 |
| `zoom` | 缩放事件 |

### 状态变化事件
| 事件名 | 说明 |
|--------|------|
| `resize` | 地图容器大小变化 |
| `center_changed` | 地图中心点变化 |
| `bounds_changed` | 可视区域范围更改 |
| `scale_changed` | 显示比例变化 |

**注意**：以上事件均无法直接判断鉴权状态。

## 五、推荐方案

### 5.1 核心逻辑

```
┌─────────────────────────────────────┐
│  地图初始化                          │
└──────────────┬──────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  劫持 console.error                   │
│  同时监听 tilesloaded                 │
└──────────────┬───────────────────────┘
               ▼
       ┌───────────────┐
       │ tilesloaded   │──────────┐
       │ 触发后延迟3秒 │          │
       └───────────────┘          │
               ▼                  ▼
       ┌───────────────┐  ┌────────────────┐
       │ 有 errorMsg?   │  │  超时10秒      │
       └───────┬───────┘  └───────┬────────┘
               │                  │
         ┌─────┴─────┐       ┌────┴────┐
         ▼           ▼       ▼         ▼
      onError     onAuth   onError   onAuth
```

### 5.2 实现代码

```typescript
const handleMapInited = (mapInstance: any) => {
  let authResolved = false
  let errorMsg = ''

  // 1. 劫持console.error捕获错误
  const originalError = console.error
  console.error = (...args: any[]) => {
    const msg = args.join(' ')
    // 检测腾讯地图的错误关键词
    if (msg.includes('鉴权') || msg.includes('KEY') || msg.includes('失败')) {
      errorMsg = msg
    }
    originalError.apply(console, args)
  }

  // 2. 监听tilesloaded事件
  mapInstance.on('tilesloaded', () => {
    // 延迟判断，等所有日志输出完毕
    setTimeout(() => {
      if (!authResolved) {
        authResolved = true
        console.error = originalError

        if (errorMsg) {
          onError?.({ detail: { errMsg: errorMsg } })
        } else {
          onAuthSuccess?.({ detail: { errMsg: 'ok' } })
        }
      }
    }, 3000)
  })

  // 3. 超时兜底（10秒）
  setTimeout(() => {
    if (!authResolved) {
      authResolved = true
      console.error = originalError

      if (errorMsg) {
        onError?.({ detail: { errMsg: errorMsg } })
      } else {
        onAuthSuccess?.({ detail: { errMsg: 'ok' } })
      }
    }
  }, 10000)
}
```

### 5.3 判断逻辑

| 条件 | 结果 |
|------|------|
| tilesloaded触发 + 有errorMsg | `onError` |
| tilesloaded触发 + 无errorMsg | `onAuthSuccess` |
| 超时10秒 + 有errorMsg | `onError` |
| 超时10秒 + 无errorMsg | `onAuthSuccess` |

## 六、已知问题

1. **错误关键词可能不全**：目前只测试了 KEY_FORMAT_ERROR，其他错误类型（日志内容未知）
2. **错误日志可能在 tilesloaded 之后才输出**：使用延迟判断解决
3. **网络错误等非控制台错误**：可能无法捕获

## 七、后续优化方向

1. 先用正确 Key 收集正常日志，再用错误 Key 收集错误日志，对比找出差异
2. 测试更多错误场景（KEY不存在、过期、配额用尽等）
3. 考虑添加网络层面的检测（script onerror）
4. 考虑添加 canvas 内容检测（检查地图是否真正渲染）
